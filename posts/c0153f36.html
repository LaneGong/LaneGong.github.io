Django ①

Django介绍

Python下有许多款不同的 Web 框架。Django是重量级选手中最有代表性的一位。许多成功的网站和APP都基于Django。Django 是一个开放源代码的 Web 应用框架，由 Python 写成。

Django本身基于 MVC 模型，即 Model（模型）+ View（视图）+ Controller（控制器）设计模式，采取了 MVT 的软件设计模式。MVT本质上和 MVC 是一样的，也是为了各组件间保持松耦合关系，只是定义上有些许不同，Django 的 MTV 分别是指：即模型（Model），视图（View）和模板（Template）。

  • M 表示模型（Model）：编写程序应有的功能，负责业务对象与数据库的映射(ORM)。
  • T 表示模板 (Template)：负责如何把页面(html)展示给用户。
  • V 表示视图（View）：负责业务逻辑，并在适当时候调用 Model和 Template。

除了以上三层之外，还需要一个 URL 分发器，它的作用是将一个个 URL 的页面请求分发给不同的 View 处理，View 再调用相应的 Model 和 Template，MTV 的响应模式如下所示：

用户操作流程：

用户通过浏览器向我们的服务器发起一个请求(request)，这个请求会去访问视图函数：

  • 如果不涉及到数据调用，那么这个时候视图函数直接返回一个模板也就是一个网页给用户。
  • 如果涉及到数据调用，那么视图函数调用模型，模型去数据库查找数据，然后逐级返回。

视图函数把返回的数据填充到模板中空格中，最后返回网页给用户。

Django安装

注：默认python环境已经搭建好了，直接开始Django

环境：MacOS （linux可类比）

学习过程中，为了方便，主要是先上手不实际开发，因此没有建立虚拟环境，直接在基础环境中安装。（后续开发具体项目时会补充上从建立virtualenv开始的步骤，其实也只是多了建环境这一步）

  1. Terminal命令：pip install django==3.2

    注：当前最新版本已经到了4.0，但django每个大版本间都会有一个LTS版本(Long Term Support)，相对稳定一些，故在这边目前采用3.2LTS版本。

    我的pip命令装的包的路径是统一到了anaconda下管理位置。

  2. Terminal cd回想要创建项目的文件夹

  3. 命令：django-admin startproject xxx创建项目

    xxx代表项目名称，eg.django-admin startproject mysite1

    django-admin是Django的一个命令行管理工具,用以对Django项目执行某些命令操作。

  4. 启动项目（测试开发阶段）

    • 进入项目文件夹mysite1
    • 执行：python3 manage.py runserver启动服务，此方式下，Django在前台启动服务，默认监听8000端口
    • 浏览器访问http://127.0.0.1:8000可看到Django启动页面
    • 若想更换端口，启动命令改为：python3 manage.py runserver 端口号

URL和视图函数

URL

URL定义：统一资源定位符(Uniform Resource Locator)，用来表示互联网上某个资源的地址。

URL一般语法格式([]表示可选)：protocol://hostname[:port]/path[?query][#fragment]

eg. http://tts.tmooc.cn:80/video/showVideo?menuld=657421&version=AID999#subject

  • protocol即通过该协议访问资源==》http（https一般在nginx里面配置）

  • hostname是指存放资源的服务器的域名系统(DNS)主机名、域名或IP地址==》tts.tmooc.cn

  • port：整数，可选，省略时使用协议的默认端口（http默认端口为80）==》:80

  • path即路由地址，由零或多个/符号隔开的字符串，一般用来表示主机上的一个目录或文件地址。路由地址决定了服务器端如何处理这个请求==》video/showVideo

  • query(查询)：用于给动态网页传递参数，可有多个参数，用&符号隔开，每个参数格式key=value==》menuld=657421&version=AID999

  • fragment(信息片段)：字符串，用于指定网络资源中的片段。有点像锚点==》#subject

Django处理URL的请求

浏览器地址栏输入：http://127.0.0.1:8000/page/2003/

  1. Django从配置文件中根据ROOT_URLCONF找到主路由文件
  2. Django加载主路由文件中的urlpatterns变量
  3. 依次匹配urlpatterns中的URL，匹配到一个合适的中断后续匹配
  4. 匹配成功：调用对应的视图函数处理请求，返回响应
  5. 匹配失败：返回404响应

视图函数

视图函数是用于接受一个浏览器请求(HttpRequest对象)并通过HttpResponse对象返回响应的函数。此函数可以接受浏览器请求并根据业务逻辑返回相应的响应内容给浏览器。

语法：

def xxx_view(request [,其他参数...]):
  return HttpResponse对象
#函数名可以自己取，参数必须要有request请求，其他可选，返回对象一定是HttpResponse对象

位置：视图函数文件views.py创建在项目同名文件夹下，然后可在其中写入视图函数。

实操

想在浏览器访问http://127.0.0.1:8000/page/2003/这个页面：

  1. 在urls.py中配置路由

    from django.contrib import admin
from django.urls import path
from . import views

#第一个是django提供的管理后台的路由
urlpatterns = [
    path('admin/', admin.site.urls),
    #http://127.0.0.1:8000/page/2003/
    path('page/2003/',views.page_2003_view)#新增此路由，若匹配上则调用page_2003_view这个视图函数处理
]

  2. 项目同名文件夹下创建views.py文件，写入该视图函数及他要处理的操作

    from django.http import HttpResponse#导入HttpResponse对象

def page_2003_view(request):
    html="<h1>这是第一个页面</h1>"
    return HttpResponse(html)

  3. 在urls.py中引入views.py文件

    from . import views

  4. 已完成，浏览器访问http://127.0.0.1:8000/page/2003/

路由配置

path()函数

导入：from django.urls import path

语法：path(route,views,name=None)

参数：

  • route：字符串类型，匹配请求的路径
  • views：指定路径所对应的视图处理函数的名称
  • name：为地址起别名，在模板中地址反向解析时使用

path转换器

语法：<转换器类型:自定义名>

作用：若转换器类型匹配到对应类型的数据，则将数据按照关键字传参的方式传递给视图函数

转换器类型 作用 样例
str 匹配除’/'之外的非空字符串 V1/users/<str:username>
int 匹配0或任何正整数，返回一个int page/<int:page>
slug 匹配任意由ASCII字母或数字以及连字符和下划线组成的短标签 detail/<slug:sl>，比如/detail/this-is-django
path 匹配非空字段，包括路径分隔符’/’ v1/users/<path:ph>，比如/v1/goods/a/b/c

实验：计算器

路由格式：http://127.0.0.1:8000/整数/操作字符串[add/sub/mul]/整数

功能：从路由中提取数据，做相应操作后返回给浏览器

#urls.py新增路由
#http://127.0.0.1:8000/整数/操作字符串[add/sub/mul]/整数
path('<int:a>/<str:op>/<int:b>',views.calculate_view)
#views.py新增视图函数
def calculate_view(request,a,op,b):
    if op not in ['add','sub','mul']:
        return HttpResponse('非法操作符!')
    result=0;
    if op=='add':
        result=a+b
    elif op=='sub':
        result=a-b
    else:
        result=a*b
    return HttpResponse('运算结果是:%s'%(result))

⚠️这边转换器中自定义名一定要和视图函数名中的形参名一致，因为是关键字传参，所以要两两匹配上，否则不一致会出现如下报错：我将自定义名改为c,e,d了，所以视图函数会获得一个未期待未定义的关键字参数c

re_path()函数

在URL的匹配过程中可以使用正则表达式进行精确匹配

语法：re_path(reg,view,name=xxx)

正则表达式采取命名分组模式，语法：(?P<name>pattern)。匹配提取参数后用关键字传参方式传递给视图函数。

示例：上述计算器例子改进，只能作两位数内运算

from django.urls import path,re_path#导入re_path

#在urlpatterns中添加re_path路由
#http://127.0.0.1:8000/整数2/操作字符串[add/sub/mul]/整数2
re_path(r'^(?P<a>\d{1,2})/(?P<op>\w+)/(?P<b>\d{1,2})$',views.calculate2_view)

#views.py
def calculate2_view(request,a,op,b):
    if op not in ['add','sub','mul']:
        return HttpResponse('非法操作符!')
    result=0;
    if op=='add':
        result=a+b
    elif op=='sub':
        result=a-b
    else:
        result=a*b
    return HttpResponse('两位数以内运算的运算结果是:%s'%(result))

⚠️通过re_path()正则读入的数据都是字符串型，关键字传参传入的也是字符串，所以像上面这么写，a,b也都是字符串而不是整型，若我输入的是12/add/12，出来的结果是1212而不是24！！

函数内部改进一下：

def calculate2_view(request,a,op,b):
    if op not in ['add','sub','mul']:
        return HttpResponse('非法操作符!')
    result=0;
    if op=='add':
        result=int(a)+int(b)
    elif op=='sub':
        result=int(a)-int(b)
    else:
        result=int(a)*int(b)
    return HttpResponse('两位数以内运算的运算结果是:%s'%(result))
