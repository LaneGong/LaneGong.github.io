CentOS7下安装MPICH及运行并行程序

摘要：

本篇博客主要是记载了在CentOS上搭建MPICH环境及运行MPI程序，同时由于学校实验课程的安排，这边还加入了一波NFS远程文件访问，不足之处，敬请指正！

CentOS下安装MPICH

  1. 更新yum库并下载相关依赖包

    yum update #更新yum库
#下载相关依赖包
yum install wget -y 
yum install gcc gcc-c++ gcc-fortran kernel-devel -y

  2. 下载安装MPI安装包

    #根目录下新建一个文件夹存放下载包
cd /
mkdir software
cd /software
wget http://www.mpich.org/static/downloads/3.4.1/mpich-3.4.1.tar.gz

tar -zxvf mpich-3.4.1.tar.gz#解压

#根目录下新建mpich下载目录
cd /
mkdir mpich-install

#进入下载包所在文件夹，编译安装
cd /software/mpich-3.4.1
./configure --prefix=/mpich-install --with-device=ch4:ofi 2>&1 | tee c.txt

    **报错：**缺少Fortran 77编译器

    解决措施：yum install gcc-gfortran

    再执行命令：

    ./configure --prefix=/mpich-install --with-device=ch4:ofi 2>&1 | tee c.txt
make
make install

  3. 配置环境

    vi ~/.bashrc
#在最底部添加下面两句命令
export MPICH=/mpich-install
export PATH=$MPICH/bin:$PATH
#更新使生效
source ~/.bashrc
#检查
mpirun --version#返回版本号则安装成功

运行MPI程序

  1. Hello world

    #include <stdio.h>
#include <mpi.h>
int main(int argc, char * argv[])
{
  int myrank;
  //初始化MPI环境
  //从这里可以看到，一般MPI指令是MPI_跟一个大写字母
	//开始的单词，后面都是小写字母。
  MPI_Init(&argc, &argv);
  // 获取当前进程在通信器MPI_COMM_WORLD中的进程号
  MPI_Comm_rank(MPI_COMM_WORLD, &myrank);
  printf("myrank = %d: Hello, world!\n", myrank);
  MPI_Finalize();
  return 0;
}

    进入程序所在文件夹：

    • mpicc mpi_test1.c -o mpi_test1编译C程序

      C++程序：mpicxx name.cpp -o name

    • mpirun -np 4 ./mpi_test1运行程序

      -np用来指定进程数，4代表给程序分配了4个进程

  2. MPI并行计算π\piπ

    #include<stdio.h>
#include<mpi.h>
int main(int argc,char *argv[])
{
    int myid,np,i,j;
    int tag=666;
    double pi=0.0; 
    double fVal;//fVal代表取Xi所对应的函数值   4/(1+x^2) 即每个矩形的高度
    int n=100;//改变n的值，表示改变切分小矩形的个数，越多越接近极限思维
    MPI_Status status;
    double h=(double)1/n; //每个矩形的宽度
    double local=0.0;//每个进程计算的面积和 
    double start,end;
    double a;
    MPI_Init(&argc,&argv);//启动并行程序    
    MPI_Comm_size(MPI_COMM_WORLD,&np);//获取进程总数，np代表进程数，在运行时可以通过 -np设置
    MPI_Comm_rank(MPI_COMM_WORLD,&myid);//获取当前进程号
    start=MPI_Wtime();//记录开始时间
    for(i=myid;i<n;i+=np) //利用np个进程同时计算各部分矩形面积
    {
            a=(i+0.5)*h;
            fVal=4.0/(1.0+a*a);//得到f(xi) 
            local+=fVal ; 
    }
    local=local*h;//得到该进程所计算的面积 
 
    //进程号！=0的进程计算的结果发送到进程0上面 
    if(myid!=0)
   {   
        MPI_Send(&local,1,MPI_DOUBLE,0,myid,MPI_COMM_WORLD); 
    }
    if(myid==0) //进程号为0就累加每个进程的计算结果 
    {
        pi=local;//得到进程0的值 后面接收就会覆盖这个值 
        for(j=1;j<np;j++)
           {
                MPI_Recv(&local,1,MPI_DOUBLE,j,j,MPI_COMM_WORLD,&status); //把其他进程的结果发送到local中 
                pi+=local;//得到所有的面积和 
           }
    }
    end=MPI_Wtime();//结束计算时间 
    if(myid==0)
        {
          printf("PI = %.15f\n",pi);
          printf("Time = %lf\n",end-start); 
          printf("np = %d\n",np);
          }
    MPI_Finalize();
    return 0;
}

利用NFS访问其他主机上MPI程序文件并运行

通过NFS去访问主要是…这是另一个实验哈哈哈，这样的话相当于是两者联动了。

主体思想：建了两台虚拟机，分别被部署为NFS的服务端和客户端，使得两台机子可以共享文件，然后将MPI程序上传至一台机子的共享文件夹中，便可在另一台机子访问这个程序并运行。

这样一来，既检验了NFS功能的实现，又实现了MPI的功能，一举两得。

关于MPICH，在两台机子上首先都按照上述安装步骤安装过了。关于部署NFS，根据后面的参考资料做即可，还是比较方便的。

  • 客户端检查NFS是否挂载成功

    最后一条显示服务端的/public挂载在客户端的/mnt/public

  • MPI程序上传至服务端共享文件夹/public

  • 在客户机上进入共享文件夹/mnt/public

    mpi2.c则是我们上传在服务端的MPI程序

  • 客户端在该文件夹下编译运行即可

多节点上运行MPI程序

前提：SSH免密登录已经实现

  • 将例子程序(cpi.c)复制到当前目录

    cp /software/mpich-3.4.1/examples/cpi* .

  • 编译mpicc –o cpi cpi.c

  • 由于想采用多节点运行即我还需要和slave1虚拟机共同工作，所以客户端这边需要修改一下NFS共享的文件夹，将原先的/mnt/public也修改成和master一样的/public，这样我在master发起的多节点并行计算就可以利用NFS直接搜索/public中的文件，否则文件夹路径不一样，容易出现找不到文件。

  • 在当前文件夹新建编辑文件servers（可改），文件的每一行为一台节点机器名，其中第一行为本机名。节点名可重复出现，表示在节点上将启动若干个进程运行，节点名后面可说明进程数。

    master:2
slave1:2

  • 关闭所有节点机器的防火墙

    systemctl stop firewalld
iptables -F
