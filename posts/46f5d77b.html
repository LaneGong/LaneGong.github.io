Birthday Paradox

Q：在一个房间里，至少有多少人，才能使其中两个人的生日是同一天的可能性超过50%？

A：答案是23人。

这就意味着在一个典型的标准小学班级（30人）中，存在两人生日相同的可能性更高。对于60或者更多的人，这种概率会大于99%。

👉🏻听上去非常不可思议，这样的一个现象被称为生日悖论。

从引起逻辑矛盾的角度来说生日悖论并不是一种悖论，它被称作悖论只是因为这个数学事实与一般直觉相抵触而已。计算与此相关的概率被称为生日问题，在这个问题之后的数学理论已被用于设计著名的密码攻击方法：生日攻击。

悖论的理论解释

通俗的理解：生日悖论会令人感到难以置信，因为人类倾向于从自己的角度看待问题。人们通常这样想，如果一个房间里加上自己共有23人，你会觉得在这22人里跟你同一天生日的可能性太低了。365天，现在却只有22个人，你可能会想概率只有22/365，所以很难在这22个人中遇上跟自己同一天生日的。其实，这是一种错误的思考方式——只是站在你自己的角度来思考有谁与你生日是一样的。

事实上：生日问题指的是在任何23个人中，两人生日相同的概率是多少，而不是你进入了一个有着22个人的房间，房间里有人会和你有相同生日的概率。

维基百科：可以把生日悖论理解成一个盲射打靶的问题。对于一个23人的房间，先考虑问题的补集：23人生日两两不同的概率是多少？为此，可以让23个人依次进入，那么每个人生日都与其他人不同的概率依次是1，364/365，363/365，362/365，361/365，等等。先进入房间的这些人生日两两不同的概率是很大的，比如说前面5个是1×364/365×363/365×362/365×361/365=97.3%。而对于最后进入房间的几个人情况就完全不同。最后几个人进入房间并且找不到同生日者的概率是… 345/365，344/365，343/365。可以把这种概率看成对一张靶的盲射：靶上有365个小格，其中有17个左右是黑格，其余是白格。假设每枪必中靶并且分布符合几何概型的话，那么连续射12枪左右任何一发都没有击中黑格的概率（投射于房间里的人生日都两两不同）是多少呢？想必大家立即会感觉到这个概率十分微小。

理解生日悖论的关键：在于考虑上述“依次进入房间”模型中最后几个进入房间的人“全部都没碰到相同生日的人”概率有多大这件事情。

简而言之，大多数人之所以会认为23人中有2人生日相同的概率应该远远小于50%，是因为将问题理解为“其他22人与你的生日相同的概率”，而非问题本意“23人之中两两之间存在生日相同”。如果考虑到这一点，直觉上会将原来的概率乘以23（注意：此算法并不正确），则会意识到概率很大了。

数学推导

假设有n个人在同一房间内，如果要计算有两个人在同一日出生的概率，在不考虑特殊因素的前提下，例如闰年，双胞胎等假设一年365日出生概率是平均分布的（现实生活中，出生概率不是平均分布的）。计算概率的方法是，首先找出$\overline{p(n)}$表示n个人中，每个人的生日日期都不同的概率。

假如n > 365，根据鸽巢原理其概率为0，假设n ≤ 365，则概率为

$\overline{p(n)}=1\times\frac{364}{365}\times\frac{363}{365}\times…\times\frac{365-n+1}{365}=\frac{365!}{365^n(365-n)!}$

==》n个人中至少2人生日相同的概率$p(n)=1-\overline{p(n)}$

通过迭代，n=23发生的概率大约是0.507。

作对比：q(n)表示房间中有n+1个人，当中与特定人（比如你）有相同生日的概率：$q(n)=1-(\frac{364}{365})^n$。当n = 22时概率只有大约0.059，约高于十七分之一。

计算机代码验证

import random
#step1:数据读入及差错判断
while True:
	m=int(input("请输入班级数m(m>=1000)："))
	if m<1000:
		print("班级数不符合要求，请重新输入！")
	else:
		while True:
			n=int(input("请输入每个班级同学的人数n："))
			if n<=0:
				print("人数不符合要求，请重新输入！")
			else:
				break
		break

#step2:利用random模块生成每个班级同学的生日并计算m个班级中存在相同生日情况的班级数 Q
Q=0
for Class in range(m):
	birth=[random.randint(1,365)for member in range(n)]
	if len(birth) != len(set(birth)):
		Q+=1

#step3:用 P=Q/M 作为对相同生日概率的估计
P=Q/m
print("{}个班级、每个班级{}人，相同生日的概率是{}".format(m,n,P))

通过控制台输入班级数(可类比为实验次数，这边要求大于等于1000)及班级人数，通过random模块随机生成一个班级同学的生日，利用集合判断是否存在相同值，并对其计数，从而得到m个班级中存在相同生日情况的班级数 Q，和总数之比便得到了存在相同生日情况的发生概率。

可以发现：在大量试验的背景下(均为1000组)，随着班级人数的改变，存在相同生日的概率也在发生变化，且概率和平常自认为的还是有很大差距的。可以观察到生日悖论的结论，在上述结果中，当班级人数为23的时候可以发现概率已经达到49.9%了。

可视化分析

单个班级（m=1即试验次数为1），观察n和至少存在2人生日相同的概率p。

目的：找到概率达到50%的点

代码：

import random
import matplotlib.pyplot as plt
import pandas as pd
from pandas import Series,DataFrame
import math
Q=0
plist=[]
for n in range(100):
	p=1-math.factorial(365)/(365**n*math.factorial(365-n))
	plist.append(p)

data=DataFrame(plist,columns=["probability"])
data['number']=range(100)
data.plot(kind="scatter",x="number",y="probability",marker="o",color="b")
plt.axhline(0.5, linestyle='--', color='red', label='50% prob')
plt.show()

结果：

由此，通过数学理论推导并结合代码可视化我们可以找到概率达到50%时人数差不多在23人。

进一步，对23这个结论进行经验性验证：

大量试验统计性分析(n=23)，生日悖论的验证，m和p的关系。

目的：以23人作约束，进行大量随机试验观察概率是否稳定在50%左右

代码：

import random
import matplotlib.pyplot as plt
import pandas as pd
from pandas import Series,DataFrame
import math
analysis={}
for num in range(1,1000):
	Q=0
	for Class in range(num):
		birth=[random.randint(1,365)for member in range(23)]
		if len(birth) != len(set(birth)):
			Q+=1
	P=Q/num
	analysis[num]=P
data1=DataFrame(list(analysis.values()),columns=["probability"])
data1['classes']=list(analysis.keys())
data1.plot(kind="scatter",x="classes",y="probability",marker="o",color="b")
plt.axhline(0.5, linestyle='--', color='red', label='50% prob')
plt.show()

结果：

通过这一步，可以清晰观察到当实验次数较少时，波动很大，不确定程度高，不具有代表性。随着实验的次数增加，存在两人生日相同情况的概率便逐渐趋向50%并保持稳定。

生日攻击

生日悖论的应用之一：生日攻击(破解密码学散列函数)

维基百科定义：是一种密码学攻击手段，所利用的是概率论中生日问题的数学原理。这种攻击手段可用于滥用两个或多个集团之间的通信。此攻击依赖于在随机攻击中的高碰撞概率和固定置换次数（鸽巢原理）。使用生日攻击，攻击者可在$\sqrt{2^n}=2^{n/2}$中找到散列函数碰撞，$2^n$为原像抗性安全性。

有点难理解👆🏻拆分开一点点看：

哈希碰撞

哈希（hash），就是将不同的输入(key)通过散列函数映射成独一无二的、固定长度的值（又称”哈希值”）。如果不同的输入得到了同一个哈希值，就发生了”哈希碰撞”（collision）。

隐患：很多网络服务通过哈希值来产生token标记用户。若两个不同用户发生了哈希碰撞，产生了相同的token，网络服务就会标记成同一个人，相当于不同的两者拥有相同的权限==》hacker attack：制造哈希碰撞，入侵系统。

生日攻击

哈希碰撞的概率取决于两个因素（假设哈希函数是可靠的，每个值的生成概率都相同）：

  • 取值空间的大小（即哈希值的长度）
  • 整个生命周期中，哈希值的计算次数

这里就要运用到生日悖论的原理了==》如果哈希值的取值空间是365，只要计算23个哈希值，就有50%的可能产生碰撞。50% 的哈希碰撞概率所需要的计算次数近似公式：$\sqrt{\frac{\pi}{2}N}$（N代表哈希的取值空间）

这种利用哈希空间不足够大，而制造碰撞的攻击方法，就被称为生日攻击（birthday attack）。

利用泰勒公式和近似结合生日悖论的数学推导便可推导出哈希碰撞概率的一般式：
$$
p(n,d)\approx1-e^{\frac{-n(n-1)}{2d}}
$$
其中，d为取值空间，n为计算次数。
