Python中的socket通信逻辑

容易混淆在一起的函数

服务端socket函数

  • bind(address)：将套接字绑定到地址，在AF_INET下，address为tuple类型(host,port)
  • listen(backlog)：开始监听TCP传入连接。backlog指定在拒绝连接之前，操作系统可以挂起的最大连接数量。该值最少为1，一般设为5
  • accept()：接受TCP连接并返回(conn,address)，其中conn是新的套接字对象，可以用来接收和发送数据，address是连接客户端的地址

客户端socket函数

  • connect(address)：连接到address处的套接字，格式认为tuple(host,port)，连接出错返回socket.error错误
  • connect_ex(address)：功能和上面相同，但成功返回0，失败会返回errno库中的errorcode中的key

公共socket函数(part)

  • recv(bufsize[,flag])：接受TCP套接字的数据，数据以字符串的形式返回，bufsize指定要接收的最大数据量
  • recvfrom(bufsize[,flag])： 接收udp套接字的数据，与recv()类似，但返回值是tuple(data,address)，其中data是包含接收数据的字符串，address是发送数据的套接字地址
  • send(string[,flag])：发送TCP数据，将string中的数据发送到连接的套接字，返回值是要发送到字节数量，该数量可能小于string的字节大小
  • sendto(string[,flag],address)：发送udp数据，将数据发送到套接字，address形式为tuple(hostaddr,port)，指定远程地址发送，返回值是发送到字节

TCP编程

客户端Client

import socket
#建立一个socket对象
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)
#AF_INET指定IPv4，SOCK_STREAM指定面向流的协议
s.connect(('www.qq.com',80))#建立连接，服务器的IP地址和端口，可以用域名自动转换成IP，且参数为tuple类型
s.send(b'GET / HTTP/1.1\r\nHost: www.qq.com\r\nConnection: close\r\n\r\n')#发送请求
#这边是向腾讯网发起连接，要求返回网页内容，使用HTTP协议，格式需符合HTTP请求
buffer=[]#利用列表建立缓冲区接收数据
while True:
	d=s.recv(1024)#利用此函数指定一次最多接收的字节数
	if d:
		buffer.append(d)
	else:#循环接收，直到返回空数据，接收完毕
		break
data=b''.join(buffer)#将列表连接成一个完整的二进制流字符串
s.close()#关闭Socket，一次完整的通信便结束了
header,html=data.split(b'\r\n\r\n',1)#分割响应头部和数据内容，碰到\r\n\r\n断开，且只分割一次
print(header.decode('utf-8'))#打印头部，解码二进制流==》utf-8
with open('qq.html','wb') as f:#将响应数据内容写入html文件
	f.write(html)

HTTPS说明：加密传输协议TLS/SSL。SSL(Secure Sockets Layer 安全套接字协议),及其继任者传输层安全（Transport Layer Security，TLS）是为网络通信提供安全及数据完整性的一种安全协议。TLS与SSL在传输层与应用层之间对网络连接进行加密。

由此对程序进行修改：

import socket
import ssl#导入ssl,ssl 模块能为底层socket连接添加SSL的支持。
s=socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s=ssl.wrap_socket(s)#从服务端获取一个CA证书,ssl.wrap_socket() 函数接受一个已存在的socket作为参数并使用SSL层来包装它
s.connect(('www.qq.com',443))#端口选择443
s.send(b'GET / HTTP/1.1\r\nHost: www.qq.com\r\nConnection: close\r\n\r\n')
buffer=[]
while True:
	d=s.recv(1024)
	if d:
		buffer.append(d)
	else:
		break
data=b''.join(buffer)
s.close()
header,html=data.split(b'\r\n\r\n',1)
print(header.decode('utf-8'))
with open('qq2.html','wb') as f:
	f.write(html)

服务端Server

  • 服务器进程首先要绑定一个端口（比如80）并监听来自其他客户端的连接。如果某个客户端连接过来了，服务器就与该客户端建立Socket连接，随后的通信就靠这个Socket连接了。
  • 由于服务器会有大量来自客户端的连接，所以服务器要能够区分一个Socket连接是和哪个客户端绑定的。一个Socket依赖4项：服务器地址、服务器端口、客户端地址、客户端端口来唯一确定一个Socket。
  • 服务器还需要同时响应多个客户端的请求，所以每个连接都需要一个新的进程或者新的线程来处理，否则，服务器一次就只能服务一个客户端了。
#Server
import socket
import threading
#线程模块，每个连接都必须创建新线程（或进程）来处理，否则，单线程在处理连接的过程中，无法接受其他客户端的连接
import time

def tcplink(sock,addr):
	print('Accept new connection from %s:%s...'%addr)
	sock.send(b'Welcome!')
	while True:
		data=sock.recv(1024)
		time.sleep(1)
		if not data or data.decode('utf-8')=='exit':
			break
		sock.send(('Hello,%s!'%data.decode('utf-8')).encode('utf-8'))
	sock.close()
	print('Connection from %s:%s closed.'%addr)

#创建基于IPv4和TCP协议的Socket
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)

#绑定监听的地址和端口
s.bind(('127.0.0.1',9999))#本机地址，外部计算机不可连
s.listen(5)#开始监听，参数用来指定等待连接的最大数量
print('Waiting for connection...')

#通过永久循环来接受来自客户端的连接
while True:
	sock,addr=s.accept()#accept()函数会等待并返回一个客户端的连接
	t=threading.Thread(target=tcplink,args=(sock,addr))#创建新线程
  #target->线程函数,args->传递给线程函数的参数，必须是tuple类型
	t.start()#启用线程活动

UDP编程

使用UDP协议时，不需要建立连接，只需要知道对方的IP地址和端口号，就可以直接发数据包–>但是不能保证到达。虽然用UDP传输数据不可靠，但它的优点是和TCP比，速度快，对于不要求可靠到达的数据，就可以使用UDP协议。

服务端Server

import socket

s=socket.socket(socket.AF_INET,socket.SOCK_DGRAM)#创建基于IPv4和UDP协议的Socket

#绑定端口
s.bind(('127.0.0.1',9999))
#udp不需要调用listen()方法而是直接接收来自任何客户端的数据
print('Bind UDP on 9999...')

while True:
	data,addr=s.recvfrom(1024)#返回数据和客户端的地址与端口
	print('Received from %s:%s.'%addr)
	s.sendto(b'Hello,%s!'%data,addr)#把数据用udp发给客户端，这边没有用多线程，可以进一步修改

客户端Client

import socket
s=socket.socket(socket.AF_INET,socket.SOCK_DGRAM)
#不需要connect()
for data in [b'Student1',b'Student2',b'Student3']:
	s.sendto(data,('127.0.0.1',9999))
	print(s.recv(1024).decode('utf-8'))
s.close()

遇到的问题

#问题1：收发不一致
#Server
if n==1:
		sock.send(('Hello,%s!'%name.decode('utf-8')).encode('utf-8'))
else:
		sock.send(b'Failure!')
    
#Client
if s.recv(1024).decode('utf-8')=='Failure!':#这边相当于已经接收过数据了
		print('用户名或密码不正确，请重新登陆！')
		continue
else:
		print(s.recv(1024).decode('utf-8'))#又接收一遍肯定为空了，没有对应的发
		break
#解决方法
message=s.recv(1024).decode('utf-8')
if message=='Failure!':
		print('用户名或密码不正确，请重新登陆！')
		continue
else:
		print(message)
		break
