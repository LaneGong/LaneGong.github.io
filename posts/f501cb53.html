下载Tomcat

下载地址：https://tomcat.apache.org/download-90.cgi

Mac用户直接选择Core中的zip或者tar.gz即可，下完之后解压。

⚠️⚠️后面这一步检查权限一定要操作检查，否则等在IDEA中部署后会出现如下报错，不能执行。

参考资料：【2021年最新】Mac版本M1安装Apache Tomcat

  • 进入下载的文件夹的bin目录，打开终端：ls -la *.sh，一般是-rw-r--r--，是没有可执行权限的

  • 执行命令：chmod +x *.sh，在这之后再执行权限查看会变成-rwxr-xr-x

  • 运行tomcat，sudo sh ./startup.sh，浏览器访问http://localhost:8080/，如下即成功

  • sudo sh ./shutdown.sh关闭tomcat

简单Servlet接口开发

还没深入学习，但通过后面简单开发，个人感觉Servlet的功能非常像Django中的view…

图片来源于Servlet 简介

  1. 创建Servlet，src/main/java/com.example.xxx右键new➡️Servlet，输入名字apitest创建

  2. 编写Servlet，在doGet中编写如下：

    @Override
   protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
       //设置同源策略，*表示通配符，允许所有ip都可以访问此接口
       response.setHeader("Access-Control-Allow-Origin","*");
       //设置都允许的请求访问类型
       response.addHeader("Access-Control-Allow-Method","POST,GET");
       //设置返回数据类型
       response.setContentType("text/html;charset=utf-8");
       response.setCharacterEncoding("utf-8");
       response.setHeader("Content-type","text/html;charset=UTF-8");
       response.getWriter().write("访问成功");
   }

  3. 路由配置/路径映射（思想和Django的路由配置非常像）

    这边采用注解配置，在Servlet类的注解中配置urlPatterns字段或者value字段

    @WebServlet(name = "apitest",urlPatterns = "/apitest")
public class apitest extends HttpServlet {}
//两种均可
@WebServlet(name = "apitest",value = "/apitest")
public class apitest extends HttpServlet {}

    启动，浏览器访问http://localhost:8080/webtest_war_exploded/apitest便可观察到访问成功字段！

  4. 更改默认访问页面，原先一开始弹出的其实是index.jsp，可以进行修改，使用web.xml配置

    <welcome-file-list>
        <welcome-file>a.html</welcome-file>
        <welcome-file>testindex.html</welcome-file>
        <welcome-file>index.jsp</welcome-file>
</welcome-file-list>

    在web.xml增加<welcome-file-list></welcome-file-list>标签配置默认页面，可以配置多个，按顺序从上到下依次查找，第一个不存在，则访问第二个，以此类推。

    在webapp文件夹下新增testindex.html文件测试，便可观察到效果。
