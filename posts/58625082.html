使用网络空间搜索引擎ZoomEye收集信息

ZoomEye介绍

全球网站都是由各种组件组成，比如用的哪种操作系统、哪种Web容器、什么服务端语言、什么Web应用等等。这些我们都认为是组件，这些组件构成了网站，形成了我们这个丰富多彩的网络世界。

而现在很多攻击都是针对网站组件进行的，拿组件下手是因为组件被使用的很多，只要研究出某个组件的漏洞，就能让使用这个组件的一大批网站沦陷。同理到主机也是一样，主机设备的搜索目前可以使用撒旦。

网站+主机构成了这个网络空间，ZoomEye的目标是采集全球这些指纹，开放搜索引擎，推进这个安全领域的进展，让更多人意识到，你暴露在网络空间上的组件或设备，实际上并不安全，藏着掖着没用！只要你在公网，ZoomEye就会默认收录，不过我们尊重不愿意搜录的网站或设备。

说明：ZoomEye和撒旦的区别是ZoomEye目前侧重于Web层面的资产发现而Shodan则侧重于主机层面。

插曲：其实原先这一part是准备着重介绍网络空间资产检索系统FOFA的。它相对于shodan来说更加本土化，拥有更多域名数据，建立了全球最大的资产规则集，并且已经更新了识别蜜罐的功能。但是！！FOFA在2022-1-20号被工信部列入了黑名单，所以目前用不了了，不知道什么时候会重新上线。

关于本网站的查询语法均可直接查看网站搜索助手文档，非常详细！（搜索前需先注册）

下文主要就实战简单展开用法和记录。网站地址：https://www.zoomeye.org

ARL资产侦查灯塔系统搭建及使用

资产侦察灯塔系统简介

ARL（Asset Reconnaissance Lighthouse）资产侦查灯塔旨在快速发现并整理企业外网资产并为资产构建基础数据库，无需登录凭证或特殊访问即可主动发现并识别资产，让甲方安全团队或者渗透测试人员快速寻找到指定企业资产中的脆弱点,降低资产被威胁利用的可能性并规避可能带来的不利影响。

**ARL资产侦查灯塔系统的思路：**资产梳理的路径会根据输入的数据进行变动与调整。如输入数据为基于域名时，采用的技术手段包括DNS 枚举、智能域名生成枚举、第三方 API 服务来获取子域名，随后将域名解析为 IP（当输入数据基于IP 时，可直接进入后续阶段），既是对 IP 进行端口扫描、服务识别、系统识别、站点识别和 WEB 指纹识别，同时爬取站点 URL，并对首页进行截图，完成资产发现。

当通过域名或是 IP 识别出对应的安全应用后，便可将其与鱼鹰(Osprey)poc 漏洞检测框架、通用漏洞扫描（CRS）、漏洞预警系统（VAS）和漏洞盒子相关联，快速准确识别资产威胁，降低资产风险。

ARL部署

  1. 拉取镜像

    命令：docker pull tophant/arl

  2. 启动 docker-compose

    git clone https://github.com/TophantTechnology/ARL
cd ARL/docker/
docker-compose up -d

    出现报错：

    根据提示先使用命令：docker volume create --name=arl_db

    再次执行：docker-compose up -d启动即可

    使用命令docker-compose ps查看运行是否正常：

  3. 打开浏览器访问：https://xx.xx.xx.xx:5003/ 中间为主机IP address

    打不开的话关一下防火墙：Advanced➡️Accept the risk and continue

    默认账号：admin

    密码：arlpass
