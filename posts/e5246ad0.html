网络路由实验

摘要：

计算机网络课实验之一，网络路由实验，主要是通过Cisco Packet Tracer网络模拟工具进行了静态路由和动态路由的配置，不足之处，欢迎指正交流！

本次实验网络模拟工具：Cisco Packet Tracer

没有官网注册账号了，直接游客登录

配置静态路由

构建网络拓扑

  • 左下角network devices ，选择3个router中的路由器，这边选择了第一个
  • 连线，左下角Connections ，不知道用什么线直接选第一个去连就行，会默认选择
  • 添加2台主机，分别连线至R7和R9
  • 修改主机成全双工模式：点击主机==》Config==》FastEthernet0==》Duplex取消auto改成full duplex

  • 配置PC2的IP，点击PC2、Desktop、IP Configuration，配置IP、子网掩码、默认网关，同理PC3

  • 配置网关(同理R9配置)

    • R7 ➡️ CLI ➡️ no
    • enable
    • conf t
    • int g0/0/1
    • ip address 13.6.10.1 255.255.255.0 根据自己实际情况设定
    • speed 100
    • duplex full
    • no shutdown
    • exit

    可见主机到第一个路由器的箭头已经从红的变成绿的了，表示这段路已经通了。

  • 配置该路由器的另一个端口(同理R9配置)

    • 继续上述窗口 int g0/0/0
    • iP address 10.1.1.2 255.255.255.0
    • speed 100
    • duplex full
    • no shutdown

  • 同样原理配置R8

配置静态路由

理论设计

对于第一个路由器(R2)

ip route 12.5.10.0 255.255.255.0 10.1.1.1
ip route 14.6.10.0 255.255.255.0 12.5.10.2

对于R1

ip route 0.0.0.0 0.0.0.0 10.1.1.2
ip route 14.6.10.0 255.255.255.0 12.5.10.2

对于R4

ip route 0.0.0.0 0.0.0.0 12.5.10.1

具体实现

根据静态路由表理论设计一台一台配置

配置R2

  • CLI(config模式下)
  • hostname R2
  • ip route 12.5.10.0 255.255.255.0 10.1.1.1
  • ip route 14.6.10.0 255.255.255.0 12.5.10.2

其余均同理。

检查

通过ping检查网络连通性

点击第一台PC、Desktop、Command Promt

ping 14.6.10.5 没有问题

配置动态路由RIP

构建网络拓扑

  • 给每个路由器添加硬件模块WIC-2T

    • 关闭路由器
    • 选择WIC-2T模块，点击后将右下角硬件拖动至上方表盘
    • 打开电源

  • 路由器连线使用DCE串口线，还是可以用第一个会自动生成

  • 配置路由IP、接口等信息，这边以一个为例

    • 进入config模式
    • int serial0/3/0
    • clock rate 64000
    • iP address 192.168.0.1 255.255.255.0
    • no shutdown
    • exit
    • int fa0/0
    • iP address 10.1.1.1 255.255.255.0
    • no shutdown
    • end

  • 配置PC，以PC0为例

    • Desktop➡️IP Configuration
    • 网关填路由器的f0/0接口的ip

RIP路由协议配置

R0

  • config模式
  • version 2
  • router rip
  • network 10.1.1.0
  • network 192.168.0.0

R1

  • config模式
  • router rip
  • version 2
  • network 10.0.0.0
  • network 192.168.0.0
  • network 192.168.1.0

R2

  • config模式
  • router rip
  • version 2
  • network 10.0.0.0
  • network 192.168.1.0

⚠️⚠️⚠️一定要给每个路由器RIP协议启用第二版，网络是不连通的

ping同网段是没问题的，但是ping PC2不同网段就会出现不可达情况，通过simulation可以发现数据包到达R0就过不去了直接返回错误信息了，改成version2就可解决。

配置动态路由OSPF

构建网络拓扑

这边就直接沿用RIP时的网络拓扑了

每个路由器在config状态下执行no router rip关闭RIP协议，其余参照上一节最后的成果。

OSPF路由协议配置

config模式下

R0：

  • router ospf 1
  • network 192.168.0.0 0.0.0.255 area 0
  • network 10.1.1.0 0.0.0.255 area 0

R1:

  • router ospf 1
  • network 192.168.0.0 0.0.0.255 area 0
  • network 192.168.1.0 0.0.0.255 area 0
  • network 10.2.1.0 0.0.0.255 area 0

R2:

  • router ospf 1
  • network 192.168.1.0 0.0.0.255 area 0
  • network 10.3.1.0 0.0.0.255 area 0
