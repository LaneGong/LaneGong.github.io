被动信息收集

什么是被动信息收集和其目的

信息收集的方式可以分为两种：

  • 被动

    被动信息收集是指利用第三方的服务对目标进行访问了解，比例：Google搜索。

  • 主动

    主动信息收集方式：通过直接访问、扫描网站，这种将流量流经网站的行为。比如：nmap扫描端口。

被动信息收集的目的：通过公开渠道，去获得目标主机的信息，从而不与目标系统直接交互，避免留下痕迹。

信息收集内容

1. IP地址段 2. 域名信息
3. 邮件地址 4. 文档图片数据
5. 公司地址 6. 公司组织架构
7. 联系电话/传真号码 8. 人员姓名/职务
9. 目标系统使⽤的技术架构 10. 公开的商业信息

信息用途

  • 信息描述目标
  • 发现目标
  • Social Engineering Attack
  • 物理缺口

DNS域名解析原理

目的：收集网站域名解析记录

DNS服务器

DNS（Domain Name Server，域名服务器）：是进行域名(domain name)和与之相对应的IP地址(IPaddress)转换的服务器，DNS中保存了一张域名(domain name)和与之相对应的IP地址 (IP address)的表，以解析消息的域名。

域名：是Internet上某一台计算机或计算机组的名称，用于在数据传输时标识计算机的电子方位（有时也指地理位置）。域名是由一串用点分隔的名字组成的，通常包含组织名，而且始终包括两到三个字母的后缀，以指明组织的类型或该域所在的国家或地区。

DNS服务器分为：

  • 根域DNS服务器

    根域DNS服务器有13个，存储了全部的顶级域名服务器的所在地址

  • 顶级域名DNS服务器

DNS查询过程

一个DNS查询过程，通过8个步骤的解析过程就使得客户端可以顺利访问www.163.com这个域名，但实际应用中，通常这个过程是非常迅速的。

客户端发起访问请求时，首先会经历如下几步：

  • 浏览器缓存。当用户通过浏览器访问某域名时，浏览器首先会在自己的缓存中查找是否有该域名对应的 IP 地址（若曾经访问过该域名且没有清空缓存便存在）
  • 系统缓存。当浏览器缓存中无域名对应 IP 则会自动检查用户计算机系统 Hosts 文件 DNS 缓存是否有该域名对应 IP
  • 路由器缓存。当浏览器及系统缓存中均无域名对应 IP 则进入路由器缓存中检查

以上三步均为客户端的 DNS 缓存 。

若都没有则会进入下述环节：

  • ISP（Internet Service Provider，互联网服务提供商）DNS 缓存(一般就是本地 DNS 服务器)：当在用户客户端查找不到域名对应 IP 地址，则将进入 ISP DNS 缓存中（本地DNS服务器）进行查询。比如你用的是电信的网络，则会进入电信的 DNS 缓存服务器中进行查找
  • 根域名服务器：当以上均未完成，则进入根服务器进行查询。全球仅有 13 台根域名服务器，1 个主根域名服务器，其余 12 为辅根域名服务器。根域名收到请求后会查看区域文件记录，若无则将其管辖范围内顶级域名（如.com）服务器 IP 告诉本地 DNS 服务器；本地DNS服务器在向顶级域名服务器发起请求
  • 顶级域名服务器：顶级域名服务器收到请求后查看区域文件记录，若无则将其管辖范围内主域名服务器的 IP 地址（163.com）告诉本地 DNS 服务器；本地DNS服务器在向主域名服务器发起请求
  • 主域名服务器：主域名服务器接受到请求后查询自己的缓存，如果没有则进入下一级域名服务器进行查找，并重复该步骤直至找到正确纪录；
  • 保存结果至缓存：本地域名服务器把返回的结果保存到缓存，以备下一次使用，同时将该结果反馈给客户端，客户端通过这个 IP 地址与 web 服务器建立链接。

DNS查询方式

  • 递归查询：客户端只发一次请求，要求对方给出最终结果。

    主机向本地域名服务器的查询一般采取递归查询。

  • 迭代查询：客户端发出一次请求，对方如果没有授权回答，它就会返回一个能解答这个查询的其它名称服务器列表，客户端会再向返回的列表中发出请求，直到找到最终负责所查域名的名称服务器，从它得到最终结果。

    本地域名服务器向根域名服务器的查询一般采取迭代查询。

DIG命令

Dig是一个在类Unix命令行模式下查询DNS包括NS记录，A记录，TXT（文本注释），MX记录，NS记录，或者任意综合查询等相关信息的工具。

命令：dig <选项1> 需要查询的域名 <选项2>

<选项1>：@DNS服务器地址：指定进行域名解析的域名服务器

<选项2>：any：显示所有类型的域名记录。默认只显示A记录

┌──(root㉿kali)-[~]
└─$ dig www.baidu.com

; <<>> DiG 9.17.21-1-Debian <<>> www.baidu.com
;; global options: +cmd
##上述部分告诉我们一些有关于dig的版本信息(version 9.17.21)和全局的设置选项，如果+nocmd在命令行下是第一个参数的话，那么这部分输出可以通过加+nocmd的方式查询出来
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 16076
;; flags: qr rd ra; QUERY: 1, ANSWER: 3, AUTHORITY: 0, ADDITIONAL: 1
##这一部分Dig告诉我们一些从DNS返回的技术信息，这段信息可以用选项 +[no]comments来控制显示，但是小心，禁止掉comments也可能关闭一些其它的选项。
;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 512
;; QUESTION SECTION:
;www.baidu.com.                 IN      A
##在这个查询段中，Dig显示出我们查询的输出，默认的查询是查询A记录，你可以显示或者禁止掉这些用+[no]question选项
;; ANSWER SECTION:
www.baidu.com.          709     IN      CNAME   www.a.shifen.com.
www.a.shifen.com.       152     IN      A       36.152.44.95
www.a.shifen.com.       152     IN      A       36.152.44.96
##我们得到我们查询的结果。www.baidu.com的地址是36.152.44.95，你可以显示或者禁止掉这些用+[no]answer选项。
;; Query time: 20 msec
;; SERVER: 114.114.114.114#53(114.114.114.114) (UDP)
;; WHEN: Mon Jan 31 12:51:22 CST 2022
;; MSG SIZE  rcvd: 101  
##最后一段默认输出包含了查询的统计数据，可以用+[no]stats保留。

子域名信息收集

子域名及其重要性

顶级域名是域名的最后一个部分，即是域名最后一点之后的字母，例如在 http://www.baidu.com 这个域名中，顶级域是.com。

常见的顶级域主要分 2 类：

  • 通用顶级类别域名常见的有：

    • 用于工商金融企业的.com；
    • 用于教育机构的.edu；
    • 用于政府部门的.gov；
    • 用于互联网络信息中心和运行中心的.net；
    • 用于非盈利组织的.org

  • 国家及地区顶级域（地理顶级域名一般由各个国家或地区负责管理）：

    • .cn 代表中国
    • .uk 代表英国
    • .jp 代表日本等

子域名（Subdomain Name）：凡顶级域名前加前缀的都是该顶级域名的子域名，而子域名根据技术的多少分为二级子域名，三级子域名以及多级子域名。

挖掘子域名的重要性：子域名是某个主域的二级域名或者多级域名，在防御措施严密情况下无法直接拿下主域，那么就可以采用迂回战术拿下子域名，然后无限靠近主域。

例如：www.xxxxx.com 主域不存在漏洞，并且防护措施严密。而二级域名 net.xxxxx.com 存在漏洞，并且防护措施松散。

  6. ksubdomain

接下去以ksubdomain作详解：

ksubdomain是一款基于无状态子域名爆破工具，支持在Windows/Linux/Mac上使用，它会很快的进行DNS爆破，在Mac和Windows上理论最大发包速度在30w/s,linux上为160w/s的速度。

下载地址：https://github.com/knownsec/ksubdomain

从releases下载二进制文件。在linux下，还需要安装libpcap-dev,在Windows下需要安装WinPcap，mac下可以直接使用。

我就直接把它下载到自己建的文件夹中并解压，Mac下便可直接使用了。打开terminal，去到该文件所在文件夹，运行命令： ./ksubdomain -d baidu.com
