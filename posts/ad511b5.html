ARPING命令

本部分可参考之前博文：ARP攻击与防御

ARP 协议是“Address Resolution Protocol”（地址解析协议）的缩写。计算机通过 ARP 协议将 IP 地址转换成 MAC 地址。

**工作原理：**在以太网中，数据传输的目标地址是 MAC 地址，一个主机要和另一个主机进行直接通信，必须要知道目标主机的 MAC 地址。计算机使用者通常只知道目标机器的 IP 信息，“地址解析”就是主机在发送帧前将目标 IP 地址转换成目标 MAC 地址的过程。简单地说，ARP 协议主要负责将局域网中的 32 位 IP 地址转换为对应的 48 位物理地址，即网卡的MAC 地址，保障通信顺利进行。

PC1想要ping IP2，首先对数据包进行封装，封装到MAC地址的时候，PC1会查询自己的ARP缓存表，发现没有IP2和其MAC地址的映射，这样的话，MAC数据帧封装失败（设备通信需要对方的MAC地址，但现在没有）。故为了获取PC2的MAC地址，PC1要发送询问信息，询问PC2的MAC地址。

Question:询问信息也需要对Mac地址封装，那现在没有，应该填什么？

规定当目的MAC地址为ff-ff-ff-ff-ff-ff时，就代表这是一个询问信息。PC2收到这个询问信息后，将这里面的IP1和MAC1添加到本地的ARP缓存表中，然后PC2发送应答信息给PC1（因为缓存表里已经有PC1的IP和MAC的映射了），这个应答信息包含PC2的IP2和MAC2。PC1收到这个应答信息，就获取了PC2的MAC地址，并添加到自己的缓存表中。

注意:目的主机先完成ARP缓存，然后才是源主机完成ARP缓存。

使用 arping 命令查看局域网中的 IP 是否有冲突

└─# arping 10.211.55.13 -c 1
ARPING 10.211.55.13
42 bytes from 00:1c:42:bc:89:0f (10.211.55.13): index=0 time=493.408 usec

--- 10.211.55.13 statistics ---
1 packets transmitted, 1 packets received,   0% unanswered (0 extra)
rtt min/avg/max/std-dev = 0.493/0.493/0.493/0.000 ms
#-c 参数表示发送的次数，我们只需要 1 次即可

对返回的结果进行一定的筛选，只取IP：

└─# arping -c 1 10.211.55.13 | grep "bytes from" | cut -d" " -f 5 | cut -d"(" -f 2 |cut -d")" -f 1
10.211.55.13

这边需要注意的是，若arping本机IP，会出现timeout问题。涉及原因：arping命令是以广播地址发送arp packets，以太网内所有的主机都会收到这个arp packets，但是本机收到之后不会Reply任何信息。因此不能在本机arping检验自己的ip。

同时arping命令只能对一个IP进行判断，若想实现网络自动扫描，需要借助脚本。

脚本内容：

#!/bin/bash
if [ "$#" -ne 1 ];then #判断用户是否输入了至少一个参数如果没有输入参数，输出提示信息并退出
	echo "Usage - ./arping.sh [interface]"
	echo "Excample - ./arping.sh eth0"
	echo "Example will perform an ARP scan of the local subnet to which eth0 is assigned"
	exit
fi
interface=$1 #将用户输入的参数传递给 interface 变量
prefix=$(ifconfig $interface | grep "inet " | cut -d 't' -f 2 | cut -d '.' -f 1-3) #获取本机 IP地址网段

#对整个网段进行 arping
for addr in $(seq 1 254);do 
	arping -c 1 $prefix.$addr | grep "bytes from" | cut -d" " -f 5 | cut -d "(" -f 2 | cut -d")" -f 1 
done

  • 在home目录新建脚本文件arping1.sh

  • 脚本文件加权限（linux中可执行程序需要拥有执行权限才可以直接执行）

    chmod +x arping1.sh

  • 执行脚本，对局域网中IP进行扫描

    └─# ./arping1.sh eth0 
10.211.55.1
10.211.55.2
10.211.55.13

在开一台虚拟机，并将两台虚拟机的IP设为一样，均为10.211.55.10

使用Netdiscover进行被动方式探测局域网中存活的机器

Netdiscover是一个主动/被动的 ARP 侦查工具。使用 Netdiscover 工具可以在网络上扫描 IP地址，检查在线主机或搜索为它们发送的 ARP 请求。

  • **主动模式：**主动的探测发现网络内主机，但是这种方式往往会引起网络管理员的注意。

    命令：netdiscover -i eth0 -r 10.211.55.1/24

    参数：

    • -i device: your network device

    • -r range: scan a given range instead of auto scan. 10.211.55.1/24或/16或/8

  • **被动模式：**被动模式的方法更加隐蔽，但是速度会比较慢，网卡被设置为混杂模式来侦听网络内的 arp 数据包进行被动式探测，这种方式就需要网络内设备发送 arp 包才能被探测到。

    命令：netdiscover -p

    参数：-p 被动模式:不发送任何东西，只进行嗅探

Hping3命令

Hping3 是一个命令行下使用的 TCP/IP 数据包组装/分析工具，通常 web 服务会用来做压力测试使用，也可以进行 DOS 攻击的实验。同样 Hping 只能每次扫描一个目标。

使用 HPING 进行压力测试：

正常网站：http://www.xuegod.cn/

对其进行压力测试：hping3 -c 1000 -d 120 -S -w 64 -p 80 --flood --rand-source xuegod.cn

参数解释：

  • -c 1000 ==>发送的数据包的数量。

  • -d 120 ==> 发送到目标机器的每个数据包的大小。单位是字节

  • -S ==> 只发送 SYN 数据包。

  • -w 64 ==> TCP 窗口大小。

  • -p 80 ==> 目的地端口(80 是 WEB 端口)。你在这里可以使用任何端口。

  • –flood ==> 尽可能快地发送数据包，不需要考虑显示入站回复。洪水攻击模式。

  • –rand-source ==> 使用随机性的源头 IP 地址。这里的伪造的 IP 地址，只是在局域中伪造。通过路由器后，还会还原成真实的 IP 地址。

  • xuegod.cn ==>目标地址

执行命令后可以发现网站会打不开，ctrl+C停止后即可恢复访问。

FPING命令

Fping 就是 ping 命令的加强版他可以对一个 IP 段进行 ping 扫描，而 ping 命令本身是不可以对网段进行扫描的。因此我们可以使用 FPING 查看局域中运行了哪些机器。

命令： fping -g 10.211.55.1/24 -c 1 > fping.txt

参数：

  • -g 表示对地址段进行扫描如果不加可以对某个 IP 进行扫描

  • -c 表示 ping 的次数

  • >fping.txt 表示将扫描的结果重定向到 fping.txt ，原因是如果扫描一个网段的话输出结果是非常多的，我们输出重定向到文件中只会获得存活的主机信息。

查看：cat fping.txt

半连接扫描

nmap扫描类型：

  • 全连接扫描，会在被扫描机器留下记录

  • 半连接扫描，不会留下记录

使用nmap进行半连接扫描：

使用nc扫描端口

nc 参数：

  • -nv 表示我们扫描的目标是个 IP 地址不做域名解析

  • -w 表示超时时间

  • -z 表示进行端口扫描
